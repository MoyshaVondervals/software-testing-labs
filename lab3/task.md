## Требования к выполнению работы: 

1.​ Тестовое покрытие должно быть сформировано на основании набора прецедентов использования сайта. 

2.​ Тестирование должно осуществляться автоматически - с помощью системы автоматизированного тестирования Selenium. 

3.​ Шаблоны тестов должны формироваться при помощи Selenium IDE и исполняться при помощи Selenium WebDriver в браузерах Firefox и Chrome. 

4.​ Предполагается, что тестируемый сайт использует динамическую генерацию элементов на странице, т.е. выбор элемента в DOM должен осуществляться не на основании его ID, а с помощью XPath.

### Прецеденты использования

### 1.​  

**Прецедент:** Поиск перевода слова 

**ID:** 1 

**Краткое описание:** Пользователь вводит слово или словосочетание для получения перевода. 

**Главный актер:** Неавторизованный/авторизованный пользователь. 

**Предусловия:** 
●​ Пользователь открыл сайт Мультитран. 
●​ Открыта вкладка по умолчанию “Словари” 

**Основной поток:** . 
1.​ Выбирает нужную языковую пару из раздела “Популярные словари” или из выпадающих меню справа от строки ввода. 
2.​ Вводит слово или словосочетание в строку поиска. 
3.​ Запускает поиск нажатием“Поиск” или клавиши “Enter”. 
4.​ Система выводит варианты перево кнопки да. 
5.​ Пользователь просматривает результаты. 

**Альтернативный поток:**  
1.​ Если слово не найдено, система предлагает варианты замены 
2.​ Пользователь уточняет запрос или меняет языковую пару. 

**Постусловия:** Пользователь получает один или несколько вариантов перевода слова. 

---

### 2.​  

**Прецедент:** Настройки сайта 

**ID:** 2 

**Краткое описание:** Пользователь открывает раздел настроек для изменения параметров работы сайта 

**Главный актер:** Авторизованный пользователь 

**Предусловия:**  
●​ Сайт Мультитран открыт. 
●​ Пользователь авторизован 

**Основной поток:**  
1.​ Пользователь переходит в раздел “Настройки”. 
2.​ Просматривает доступные параметры. 
3.​ Изменяет нужные настройки используя checkbox. 
4.​ Сохраняет изменения, нажатием кнопки “Сохранить” или клавиши “Enter”. 
5.​ Система применяет новые параметры работы. 

**Альтернативный поток:**  
1.​ Если изменение настройки недоступно, система оставляет прежние параметры. 
2.​ Если пользователь покидает страницу настроек, система не сохраняет новые значения. 

**Постусловия:** Новые настройки пользователя сохранены и применены. 
Либо настройки остаются без изменений, если пользователь покинул страницу.

### 3.​  

**Прецедент:** Создание темы на форуме 

**ID:** 3 

**Краткое описание:** Пользователь создает тему на форуме. 

**Главный актер:** Авторизованный пользователь 

**Предусловия:**  
●​ Пользователь прошел авторизацию и находится на странице с созданием темы на форуме. 

**Основной поток:**  
1.​ Пользователь вводит название темы в поле “Тема:”. 
2.​ Пользователь вводит текст в TextArea “Сообщение:”. 
3.​ Пользователь нажимает “Предварительный просмотр”. 
4.​ Пользователь перенаправляется на страницу, где он видит сообщение таким, каким оно будет отображаться на форуме. 
5.​ Пользователь нажимает “Сохранить сообщение”. 

**Альтернативный поток:**  
●​ Дополнительно, в форме для создания темы пользователь может отметить checkbox “Уведомлять о новых ответах по почте”. 
●​ Дополнительно, в форме для создания темы пользователь может отметить checkbox “Проверка орфографии”, для проверки орфографии в его сообщении. 
●​ Пользователь может изменить текст сообщения и снова нажать “Предварительный просмотр”. После чего повторяются 8 и 9 операции из основного потока. 

**Постусловия:** В случае, если пользователь нажал “Сохранить сообщение”, то на форуме его сообщение создается в определенной языковой категории, и становится видимым для всех авторизованных пользователей.

### 4.​  

**Прецедент:** Поиск слов, содержащих заданные буквы. 

**ID:** 4 

**Краткое описание:** Пользователь применяет специальные символы для поиска слова по шаблону. 

**Главный актер:** Неавторизованный/авторизованный пользователь. 

**Предусловия:**  
●​ Пользователь открыл раздел “Поиск слов, содержащих заданные буквы.” в разделе “Словари” 
●​ Пользователь прочитал принцип использования символов * и ?. 

**Основной поток:**  
1.​ Пользователь вводит шаблон слова с масками в TextArea  
2.​ Устанавливает дополнительные параметры поиска через checkbox. 
3.​ Запускает поиск нажатием кнопки “Поиск” или клавиши “Enter”. 
4.​ Система выводит слова, соответствующие шаблону. 
5.​ Пользователь выбирает нужное слово и просматривает перевод. 

**Альтернативный поток:**  
1.​ Если шаблон задан слишком широко, система выводит большой список результатов или сообщение “Ничего не найдено”. 
2.​ Пользователь сужает шаблон и повторяет поиск. 

**Постусловия:** Пользователь получает список слов, соответствующих заданному шаблону.

### 9.​  

**Прецедент:** Просмотр вопросов/ответов пользователя. 

**ID:** 9 

**Краткое описание:** Пользователь видит кол-во вопросов/ответов любого пользователя за все время, может перейти и посмотреть их. 

**Главный актер:** Неавторизованный/авторизованный пользователь. 

**Предусловия:** Пользователь находится на странице профиля другого пользователя. 

**Основной поток:**  
1.​ Пользователь видит раздел “Участие в форумах” 
2.​ Пользователь видит кол-во вопросов и ответов пользователя, разделенное по языкам

●​ В случае, если у пользователя нет истории с вопросами и ответами, то на месте с кол-вом вопросов и ответов отображается “-”. 

**Постусловия:**  
●​ Пользователь получил доступ ко всем ответам и вопросам другого пользователя

### 10.​  

**Прецедент:** Добавление нового перевода 

**ID:** 10 

**Краткое описание:** Пользователь добавляет новый перевод слова или фразы в определенный словарь. 

**Главный актер:** Авторизованный пользователь 

**Предусловия:**  
Пользователь прошел авторизацию и находится на странице словаря. 
Пользователь находится на странице добавления нового перевода 

**Основной поток:**  
1.​ Пользователь переходит на страницу с формой для добавления перевода. 
2.​ Пользователь вводит слово/фразу на языке словаря. 
3.​ Пользователь вводит перевод слова/фразы на нужном языке. 
4.​ Пользователь нажимает “Сохранить” 

**Альтернативный поток:**  
●​ В случае нескольких переводов, пользователь вводит их через символ “;”. 
●​ Пользователь может выбрать тематику перед добавлением. 
●​ В случае если слова может не существовать, то пользователь должен либо подтвердить, что такое слово есть, нажав “Сохранить статью”, либо отменить действие, нажав “Назад”. 

**Постусловия:** Пользователь успешно добавил новый перевод слова или фразы в определенный словарь.
